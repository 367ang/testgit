# Asahi Linux 双系统安装指南

本文档详细说明了在保留现有 macOS 15.7.2 的前提下，为 M1 Pro MacBook Pro 添加 Asahi Linux 系统的完整方案。方案采用官方 Asahi Linux 安装器实现，确保安装过程安全、自动化且可靠。

> [!success] 已获用户批准
> 此设计方案已获得用户明确接受（方案A：使用官方 Asahi Linux 安装器）

## 系统需求

### 硬件要求
- Apple MacBook Pro with M1 Pro chip（安装器会自动验证）
- 已连接电源适配器（全程必需）

### 软件要求
- macOS 15.7.2 或更高版本
- 稳定的互联网连接（用于下载安装文件）

### 存储要求
- macOS 分区需有 **至少 128GB + 20GB 余量** 的可用空间
  - 128GB 用于 Asahi Linux 系统
  - 额外 20GB 用于分区调整过程中的临时空间

## 安装方案概述

我们采用 **方案A：官方 Asahi Linux 安装器**，理由如下：

1. **官方支持**：专为 Apple Silicon 设计的安装器，经 M 系列芯片优化和测试
2. **高度自动化**：全程自动处理分区调整、系统安装和引导配置
3. **错误最小化**：减少用户手动操作，降低失误风险
4. **完整兼容性**：自动处理苹果安全特性（如 SIP）的临时调整
5. **官方文档支持**：遇到问题时有最完善的参考资源

## 详细实施步骤

### 前期准备
> [!warning] 重要前置步骤
> 1. **数据备份**：使用 Time Machine 完整备份重要数据（虽然过程安全，但作为预防措施）
> 2. **电源连接**：确保 MacBook Pro 全程连接电源适配器
> 3. **网络准备**：准备稳定的 WiFi 或有线网络连接（需下载约 500MB-1GB 文件）
> 4. **空间检查**：通过 "关于本机" → "存储" 确认有足够可用空间

### 步骤1：获取安装器
1. 访问官方网站：https://asahi.linux
2. 下载最新的 Asahi Linux 安装器（macOS 版本）
3. （可选）验证下载文件的 SHA256 校验和

### 步骤2：运行安装器
1. 双击运行下载的安装器应用程序
2. 安装器将自动：
   - 检测硬件型号（确认是 M1 Pro）
   - 扫描当前 APFS 容器结构
   - 显示 macOS 当前分区布局和可用空间

### 步骤3：分区调整（自动执行）
安装器将自动完成以下操作：
> [!note] 自动分区过程
> - 安全缩减现有 APFS 容器以腾出空间
> - 在腾出的空间中创建新的 APFS 容器用于 Asahi Linux
> - 在新容器中创建必要卷：
>   - `Asahi Linux`：根文件系统卷（约 120GB）
>   - `Asahi Linux Recovery`：恢复卷（约 2GB）
>   - `EFI`：引导分卷（小容量）
> - **此过程保持原有 macOS 分区完整不变**

### 步骤4：系统文件下载和安装
安装器将：
1. 从 asahi.linux 官方服务器下载：
   - 基础系统镜像（Alpine Linux 基础，针对 Apple Silicon 优化）
   - 内核和固件包（包含 M1 Pro 专用驱动）
   - 默认桌面环境（GNOME）
   - 必要的工具链和基本应用
2. 显示下载进度条和速度
3. 验证下载文件的完整性
4. 解压并写入之前创建的分区

### 步骤5：引导配置
安装器将：
1. 修改引导加载程序
2. 添加 Asahi Linux 启动条目到引导菜单
3. 保留原有 macOS 启动条目
4. 设置默认启动项（推荐保持 macOS 为默认以减少干扰）
5. 将配置持久化

### 步骤6：安装完成和重启
1. 安装器提示重启电脑
2. 重启后将看到引导菜单，选项包括：
   - macOS（默认选项，若 3秒内无操作自动启动）
   - Asahi Linux
   - Asahi Linux Recovery（用于故障恢复）
   - 固件安全选项（取决于具体 Mac 型号）

## 预计时间表
- **准备阶段**：10-15 分钟
- **安装器运行**：20-40 分钟（主要取决于网络下载速度）
  - 分区调整：5-10 分钟
  - 文件下载：10-30 分钟
  - 系统写入：5-10 分钟
  - 引导配置：2-5 分钟
- **首次启动和设置**：5-10 分钟
- **总计**：约 40-75 分钟（不包括备份时间）

## 验证和测试步骤
安装完成后建议执行以下验证：

> [!tip] 验证清单
> 1. **引导菜单验证**：确认能看到双系统选项并正常切换
> 2. **macOS 验证**：重新进入 macOS，检查系统完整性和数据
> 3. **Asahi Linux 验证**：
>    - 登录默认桌面环境（GNOME）
>    - 检查分区大小（应显示约 128GB 可用）
>    - 测试基本功能：网络、声音、显示
>    - 更新系统：`sudo apk update && sudo apk upgrade`
> 4. **性能基准**：简单测试 CPU 和内存使用情况

## 错误处理和恢复机制

> [!warning] 常见问题及解决方案
> - **分区调整失败**：安装器会自动回滚到原始状态，保持 macOS 完整
> - **下载中断**：支持断点续传，网络恢复后自动继续
> - **写入错误**：安装器会验证写入完整性，失败时清理已写入数据并报告具体错误
> - **引导配置问题**：提供恢复模式，可通过 macOS 恢复引导 Linux 进行修复
> - **安装中途取消**：在写入阶段前可安全取消，仅删除临时下载文件

## 后续维护注意事项

### 系统更新
> [!note] 更新方法
> - **macOS**：通过 "系统偏好设置" → "软件更新" 进行普通更新
> - **Asahi Linux**：使用终端命令更新
>   ```bash
>   sudo apk update && sudo apk upgrade
>   ```

### 空间管理
- 安装过程会预留少量余量
- 日常使用建议为每个系统保持至少 10% 以上空闲空间
- 可通过定期清理缓存和卸载不需要的应用来管理空间

### 备份策略
> [!tip] 建议的备份计划
> - **macOS**：继续使用 Time Machine 进行定期备份
> - **Asahi Linux**：考虑使用外部硬盘或云服务定期备份重要数据
> - **重要文件**：建议保持关键文件的多份副本

### 恢复准备
- 熟悉如何通过按键（开机时按住 Option 键）进入启动管理器
- 了解如何通过 macOS 恢复模式（开机时按住 Command+R）访问修复工具
- 知道 Asahi Linux 恢复分区的用途和访问方法
