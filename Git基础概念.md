# Git 基础概念

Git 是一个分布式版本控制系统，理解其核心概念是高效使用的关键。

## 三大核心区域

### 工作区 (Working Directory)

工作区是你实际编辑文件的地方，就是你看到的项目目录。

**实际应用场景：**
- 编辑代码文件
- 创建新文件
- 删除不需要的文件

```bash
# 查看工作区状态
git status
```

### 暂存区 (Staging Area)

暂存区是临时存放待提交文件的地方，相当于一个"预提交"区域。

**实际应用场景：**
- 选择性地提交部分修改
- 组织提交内容，使每次提交有意义

```bash
# 将工作区的修改添加到暂存区
git add <file>      # 添加单个文件
git add .           # 添加所有修改
git add -p          # 交互式添加部分修改
```

### 本地仓库 (Local Repository)

本地仓库存储提交历史，每个提交都是一个完整的项目快照。

**实际应用场景：**
- 保存项目历史版本
- 创建分支进行功能开发
- 查看历史提交记录

```bash
# 将暂存区内容提交到本地仓库
git commit -m "提交信息"

# 查看提交历史
git log --oneline
```

### 远程仓库 (Remote Repository)

远程仓库托管在服务器上，用于团队协作和备份。

**实际应用场景：**
- 多人协作开发
- 代码备份
- CI/CD 集成

```bash
# 添加远程仓库
git remote add origin <url>

# 推送到远程仓库
git push origin main
```

## 工作流程图解

```
工作区 → git add → 暂存区 → git commit → 本地仓库 → git push → 远程仓库
```

## 为什么要有暂存区？

暂存区的存在让你可以：

1. **精细控制提交内容** - 不必一次提交所有修改
2. **修改提交** - 提交前可以继续修改已暂存的文件
3. **部分提交** - 将不同功能的修改分开提交

**示例：部分提交**

```bash
# 场景：你同时修改了 bug 修复和新功能
# 希望分开提交

# 只提交 bug 修复的文件
git add bug-fix.js
git commit -m "fix: 修复登录验证问题"

# 再提交新功能
git add new-feature.js
git commit -m "feat: 添加用户头像功能"
```

## 对象模型

Git 使用三种对象存储数据：

| 对象类型 | 说明 | 示例 |
|---------|------|------|
| Blob | 存储文件内容 | 每个文件版本对应一个 blob |
| Tree | 存储目录结构 | 包含文件名和 blob 引用 |
| Commit | 存储提交信息 | 包含 tree、作者、时间、父提交 |

> [!tip]
> 理解 Git 的对象模型有助于理解为什么 Git 处理分支和合并如此高效
